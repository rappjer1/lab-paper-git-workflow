# Schema Reference

Paper Scaffold uses small user-authored YAML files and generated JSON reports. The schemas are intentionally conservative and dependency-free.

The CLI exposes the schema names used in this document:

```bash
paper-scaffold schema list
paper-scaffold schema show artifact-manifest
paper-scaffold schema show artifact-manifest --json
```

User-authored schemas should remain backward-compatible after v1.0. Generated JSON files may add fields, but patch and minor releases should not silently remove fields or change field meaning.

## User-Authored Schemas

### artifact-manifest

File: `metadata/artifact_manifest.yaml`

Purpose: list selected manuscript artifacts and their provenance.

Minimal example:

```yaml
artifacts:
  - id: example_metric_plot
    type: figure
    manuscript_path: figures/example_metric_plot.pdf
    source_repo: ../research_project
    source_path: outputs/example_metric_plot.pdf
```

Required fields:

- top-level `artifacts`
- artifact `id`
- artifact `type`
- artifact `manuscript_path`
- artifact `source_path`

Optional fields:

- `source_repo`
- `generated_by`
- `input_data`
- `last_updated`
- `caption_hint`
- `status`

Compatibility notes:

- Supported artifact types include `figure`, `table`, `data_summary`, `supplement_figure`, and `supplement_table`.
- New optional artifact metadata fields may be accepted later.
- Existing required field names should remain stable after v1.0.

### manuscript-config

File: `metadata/manuscript_config.yaml`

Purpose: configure manuscript paths and validation thresholds.

Minimal example:

```yaml
project:
  title: Example Paper
  slug: example_paper
  main_tex: main.tex
  supplement_tex: supplement/supplement.tex
  has_supplement: true
validation:
  max_file_size_mb: 25
  forbidden_patterns:
    - "*.npz"
    - "*.pt"
```

Required fields:

- none; Paper Scaffold supplies defaults when this file or individual fields are missing.

Optional fields:

- `project.title`
- `project.slug`
- `project.main_tex`
- `project.supplement_tex`
- `project.has_supplement`
- `validation.max_file_size_mb`
- `validation.forbidden_patterns`

Compatibility notes:

- New optional config fields may be added.
- Defaults should preserve existing manuscript repositories after v1.0.

### terminology-map

File: `metadata/terminology_map.yaml`

Purpose: map implementation terms to publication-facing wording and flag banned terms.

Minimal example:

```yaml
terms:
  internal_metric_name:
    publication_label: descriptive metric name
    banned_in:
      - main_text
    allowed_contexts:
      - supplement_provenance
```

Required fields:

- top-level `terms` when the file is present.

Optional fields:

- term `publication_label`
- term `banned_in`
- term `allowed_contexts`

Compatibility notes:

- Missing terminology maps are warnings, not errors.
- New optional per-term metadata may be accepted later.

## Generated Schemas

### validation-report

File: `validation_report.json`

Generated by: `validate --write-json`

Purpose: machine-readable validation diagnostics.

Minimal example:

```json
{
  "tool": "Paper Scaffold",
  "version": "0.9.9",
  "timestamp": "2026-01-01T00:00:00+00:00",
  "path": "./paper",
  "summary": {
    "errors": 0,
    "warnings": 0,
    "info": 1
  },
  "diagnostics": []
}
```

Required fields:

- `tool`
- `version`
- `timestamp`
- `path`
- `summary`
- `diagnostics`

Optional fields:

- future generated metadata fields

Compatibility notes:

- New generated fields may be added.
- Existing documented fields should not be silently removed after v1.0.

### provenance-ledger

File: `metadata/provenance_ledger.json`

Generated by: `provenance-report --write-json`

Purpose: machine-readable bill of materials for manuscript artifacts.

Minimal example:

```json
{
  "tool": "Paper Scaffold",
  "version": "0.9.9",
  "generated_at": "2026-01-01T00:00:00+00:00",
  "manuscript_repo": "./paper",
  "summary": {
    "total_manifest_artifacts": 1,
    "current": 1
  },
  "artifacts": []
}
```

Required fields:

- `tool`
- `version`
- `generated_at`
- `manuscript_repo`
- `summary`
- `artifacts`

Optional fields:

- additional artifact details and future generated metadata fields

Compatibility notes:

- Artifact entries may include hashes, mtimes, source paths, manuscript paths, usage files, and status.
- New fields may be added to support richer provenance.

### artifact-lock

File: `metadata/artifact_lock.json`

Generated by: `freeze-artifacts`

Purpose: freeze manuscript artifact hashes at submission or revision handoff.

Minimal example:

```json
{
  "tool": "Paper Scaffold",
  "version": "0.9.9",
  "generated_at": "2026-01-01T00:00:00+00:00",
  "manuscript_repo": "./paper",
  "artifacts": [
    {
      "artifact_id": "example_metric_plot",
      "artifact_type": "figure",
      "manuscript_path": "figures/example_metric_plot.pdf",
      "manuscript_sha256": "..."
    }
  ]
}
```

Required fields:

- `tool`
- `version`
- `generated_at`
- `manuscript_repo`
- `artifacts`

Optional fields:

- future generated metadata fields

Compatibility notes:

- New fields may be added to lock entries.
- Current artifact identifiers and manuscript hashes should remain meaningful after v1.0.

### lock-comparison

File: `lock_comparison.json`

Generated by: `compare-lock --write-json`

Purpose: compare current manuscript artifacts to a lock.

Minimal example:

```json
{
  "tool": "Paper Scaffold",
  "created_at": "2026-01-01T00:00:00+00:00",
  "manuscript_repo": "./paper",
  "lock_path": "./paper/metadata/artifact_lock.json",
  "summary": {
    "locked_artifacts": 1,
    "unchanged": 1,
    "changed": 0,
    "missing": 0,
    "new": 0
  },
  "changed": [],
  "missing": [],
  "new": []
}
```

Required fields:

- `tool`
- `created_at`
- `manuscript_repo`
- `lock_path`
- `summary`
- `changed`
- `missing`
- `new`

Optional fields:

- future generated metadata fields

Compatibility notes:

- New comparison categories may be added only in a compatible way.
- Existing summary fields should remain stable after v1.0.

## Stability

These schemas are pre-1.0 stable in v0.9.9. The v1.0 release should freeze required field names, diagnostic code names, and generated JSON shapes for normal manuscript workflows.

# Schema Reference

Paper Scaffold uses small user-authored YAML files and generated JSON reports. The schemas are intentionally conservative and dependency-free.

The CLI can print schema summaries:

```bash
paper-scaffold schema list
paper-scaffold schema show artifact-manifest
paper-scaffold schema show artifact-manifest --json
```

## artifact_manifest.yaml

Source: user-authored.

Purpose: list selected manuscript artifacts and their provenance.

Minimal valid example:

```yaml
artifacts:
  - id: example_metric_plot
    type: figure
    manuscript_path: figures/example_metric_plot.pdf
    source_repo: ../research_project
    source_path: outputs/example_metric_plot.pdf
```

Required top-level field: `artifacts`.

Required artifact fields: `id`, `type`, `manuscript_path`, `source_path`.

Supported artifact types: `figure`, `table`, `data_summary`, `supplement_figure`, `supplement_table`.

## manuscript_config.yaml

Source: user-authored, with defaults if missing.

Purpose: configure manuscript paths and validation thresholds.

```yaml
project:
  title: Example Paper
  slug: example_paper
  main_tex: main.tex
  supplement_tex: supplement/supplement.tex
  has_supplement: true
validation:
  max_file_size_mb: 25
  forbidden_patterns:
    - "*.npz"
    - "*.pt"
```

All fields are optional; Paper Scaffold supplies defaults.

## terminology_map.yaml

Source: user-authored.

Purpose: map implementation terms to publication-facing wording and flag banned terms.

```yaml
terms:
  internal_metric_name:
    publication_label: descriptive metric name
    banned_in:
      - main_text
    allowed_contexts:
      - supplement_provenance
```

## provenance_ledger.json

Source: generated by `provenance-report --write-json`.

Purpose: machine-readable bill of materials for manuscript artifacts.

Minimal shape:

```json
{
  "tool": "Paper Scaffold",
  "version": "0.9.0",
  "generated_at": "2026-01-01T00:00:00+00:00",
  "manuscript_repo": "./paper",
  "summary": {
    "total_manifest_artifacts": 1,
    "current": 1
  },
  "artifacts": []
}
```

Generated artifact entries include hashes, mtimes, source paths, manuscript paths, usage files, and status.

## artifact_lock.json

Source: generated by `freeze-artifacts`.

Purpose: freeze manuscript artifact hashes at submission or revision handoff.

```json
{
  "tool": "Paper Scaffold",
  "version": "0.9.0",
  "generated_at": "2026-01-01T00:00:00+00:00",
  "manuscript_repo": "./paper",
  "artifacts": [
    {
      "artifact_id": "example_metric_plot",
      "artifact_type": "figure",
      "manuscript_path": "figures/example_metric_plot.pdf",
      "manuscript_sha256": "..."
    }
  ]
}
```

## validation_report.json

Source: generated by `validate --write-json`.

Purpose: machine-readable validation diagnostics.

```json
{
  "tool": "Paper Scaffold",
  "version": "0.9.0",
  "timestamp": "2026-01-01T00:00:00+00:00",
  "path": "./paper",
  "summary": {
    "errors": 0,
    "warnings": 0,
    "info": 1
  },
  "diagnostics": []
}
```

## lock_comparison.json

Source: generated by `compare-lock --write-json`.

Purpose: compare current manuscript artifacts to a lock.

```json
{
  "tool": "Paper Scaffold",
  "created_at": "2026-01-01T00:00:00+00:00",
  "manuscript_repo": "./paper",
  "lock_path": "./paper/metadata/artifact_lock.json",
  "summary": {
    "locked_artifacts": 1,
    "unchanged": 1,
    "changed": 0,
    "missing": 0,
    "new": 0
  },
  "changed": [],
  "missing": [],
  "new": []
}
```

## Stability

These schemas are pre-1.0 stable in v0.9. The 1.0 release should freeze required field names, diagnostic code names, and generated JSON shapes for normal manuscript workflows.

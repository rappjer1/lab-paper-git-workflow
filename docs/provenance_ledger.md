# Provenance Ledger

Paper Scaffold provenance means a generated bill of materials for manuscript figures and tables. It connects the user-authored `metadata/artifact_manifest.yaml` to the files currently present in the manuscript repository and to the TeX files that reference them.

It answers practical manuscript questions:

- Which figures and tables are listed in the manifest?
- Do the manuscript copies exist?
- Do the recorded source files still exist?
- Do source and manuscript hashes match?
- Which TeX files reference each artifact?
- Are there figure/table files in the manuscript repo that are not listed in the manifest?
- Are TeX files referencing missing artifacts?

## What It Does Not Guarantee

The provenance ledger does not prove the science is correct. It does not rerun analyses, compile LaTeX, validate numerical methods, or replace review by the authors. It also cannot prove that a source file was generated by the stated script; it records the declared source metadata and compares available file hashes.

## Provenance Versus Raw Reproducibility

Raw reproducibility asks whether a full analysis can be rerun from raw inputs to final outputs. Paper Scaffold provenance is narrower: it checks whether the manuscript artifacts in a clean paper repo can be traced back to declared source files and whether the manuscript currently references them.

This is useful before submission and during revisions even when full computational reproducibility lives in a separate research repository, archive, or workflow system.

## Commands

Generate a Markdown report:

```bash
paper-scaffold provenance-report --manuscript-repo ./paper --write-md ./paper/provenance_report.md
```

Generate a machine-readable ledger:

```bash
paper-scaffold provenance-report --manuscript-repo ./paper --write-json ./paper/metadata/provenance_ledger.json
```

Print compact status counts:

```bash
paper-scaffold artifact-status --manuscript-repo ./paper
```

Freeze current manuscript artifact hashes:

```bash
paper-scaffold freeze-artifacts --manuscript-repo ./paper --write-lock ./paper/metadata/artifact_lock.json
```

Compare current artifacts against a previous lock:

```bash
paper-scaffold compare-lock --manuscript-repo ./paper --lock metadata/artifact_lock.json --write-report ./paper/lock_comparison.md
```

## Generated Ledger Fields

Each generated artifact entry includes:

- `artifact_id`
- `artifact_type`
- `manuscript_path`
- `source_path`
- `source_repo`
- `source_git_commit`
- `source_exists`
- `manuscript_exists`
- `source_sha256`
- `manuscript_sha256`
- `source_mtime`
- `manuscript_mtime`
- `copied_at`
- `generated_by`
- `used_in_tex_files`
- `used_in_main_or_supplement`
- `status`
- `notes`

Statuses are:

- `current`: source and manuscript copies exist and hashes match.
- `stale`: source and manuscript copies exist but hashes differ.
- `missing_source`: the manifest declares a source, but the source file is missing.
- `missing_manuscript`: the manifest declares a manuscript artifact, but the manuscript copy is missing.
- `untracked`: a manuscript figure/table exists but is not listed in the manifest.
- `unknown`: the manuscript copy exists but no usable source was declared.

## Revision Use

During revisions, generate a provenance report before and after updating figures or tables. Review `stale`, `missing_source`, and `untracked` entries before committing. Use `freeze-artifacts` after a submission or response handoff to record the exact hashes of manuscript artifacts at that point, then use `compare-lock` to see which locked artifacts changed or disappeared.

## Reviewer Responses

For reviewer responses, provenance reports help show which response figures or tables are currently included and whether they trace back to declared source outputs. Keep confidential review text out of public repositories, but keep artifact provenance clean.

## Before Submission

Before submission:

```bash
paper-scaffold release-check --manuscript-repo ./paper --write-report ./paper/release_check.md
paper-scaffold provenance-report --manuscript-repo ./paper --write-md ./paper/provenance_report.md --write-json ./paper/metadata/provenance_ledger.json
paper-scaffold freeze-artifacts --manuscript-repo ./paper --write-lock ./paper/metadata/artifact_lock.json
paper-scaffold compare-lock --manuscript-repo ./paper --lock metadata/artifact_lock.json --write-report ./paper/lock_comparison.md
paper-scaffold package-submission --manuscript-repo ./paper --output ./submission_package
```

Review all warnings and errors before sharing or syncing the repository.

## Example Snippet

```text
Artifact status
- total manifest artifacts: 3
- current: 3
- stale: 0
- missing source: 0
- missing manuscript artifact: 0
- untracked manuscript artifacts: 0
- unreferenced artifacts: 0
- referenced missing artifacts: 0
```

The Markdown report also includes current artifacts, stale artifacts, missing sources, missing manuscript files, untracked artifacts, missing TeX references, a usage map, and recommended next actions.
